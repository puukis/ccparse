# Support

## Before Opening an Issue

- read the `README.md`
- run `npm run check`
- verify behavior against the fixtures with `npm run fixtures:validate`
- check whether the issue is about raw Claude Code data shape rather than library behavior

## Good Issue Reports

A useful support or bug report usually includes:

- the `ccparse` version or commit
- Node version
- the API call you made
- the warning codes you saw, if any
- whether the problem reproduces with a redacted fixture

## Usage Questions

For usage questions, open a GitHub issue with enough context to reproduce the behavior. If the question depends on sensitive local transcript data, reduce it to a minimal redacted example where possible.
