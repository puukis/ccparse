# Security Policy

## Supported Versions

The current development line is the only version actively maintained for security fixes until the project adopts a formal release support policy.

| Version | Supported |
| --- | --- |
| 0.1.x | Yes |
| < 0.1.0 | No |

## Reporting a Vulnerability

Please do not open a public issue for security-sensitive bugs.

If the repository has GitHub Security Advisories enabled, use that private reporting flow. Otherwise, contact the project maintainers privately before public disclosure and include:

- affected version or commit
- impact summary
- reproduction details
- any proof-of-concept or logs needed to validate the issue

Maintainers will acknowledge confirmed reports, assess severity, and coordinate a fix and disclosure plan.
